이음 대시보드 재인입/작업자 최종 수정

반영내역
1. 재인입 로직 충돌 수정
- 기존: 같은 연락처가 다시 들어오면 중복으로 빠질 수 있었음
- 수정: 같은 연락처 + 같은 DB단계 + 같은 날짜만 중복 제외
- 같은 연락처 + 같은 DB단계 + 다른 날짜는 1차 재인입/2차 재인입으로 집계 가능

2. 엑셀 파서 중복 기준 수정
- 기존 업로드 파일 내부에서 같은 전화번호가 있으면 뒤 건을 제거
- 수정: 같은 전화번호라도 날짜가 다르면 유지
- 재인입 데이터가 업로드 단계에서 사라지는 문제 보정

3. 작업자 연동 보강
- 컨설팅리스트 2차 RAW의 작업자/처리자/상담원/담당자/접수자/등록자 값을 operator로 반영
- DB관리 작업자 칼럼에 표시
- 기존 RAW에 값이 남아 있으면 조회 시 보정

4. DB관리 작업자 필터 추가
- 전체 작업자 / 임선진 / 정선영 / 시스템 등 작업자별 필터 가능

5. DB관리 매체/상세매체 표시 정리
- TU-알바리치: 매체 TU / 상세 알바리치
- 휴그린-메일: 매체 휴그린 / 상세 메일
- 인바운드-인입콜: 매체 인바운드 / 상세 인입콜

6. 상담결과 중복 옵션 유지

적용방법
1. ZIP 압축 해제
2. 문서/GitHub/ieum-dashboard 폴더에 덮어쓰기
3. npm run dev 테스트
4. GitHub Desktop Commit
5. Push origin
6. Vercel 자동배포 확인

중요
- Apps Script 변경 없음
- 기존 업로드 데이터 중 이미 재인입이 누락된 건은 엑셀 파서 단계에서 사라졌을 수 있음
- 재인입을 정확히 복구하려면 FIRST_DB_RAW / SECOND_DB_RAW / DASHBOARD_LEADS 2행 이하 삭제 후 컨설팅UTM → 컨설팅리스트 순서로 재업로드 권장
